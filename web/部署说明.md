## 🚀 自动部署 (推荐)

项目已配置 GitHub Actions，推送到 main 分支后自动部署到 GitHub Pages。

### 部署流程

1. **推送代码到 main 分支**
   ```bash
   git add .
   git commit -m "feat: 更新内容"
   git push
   ```

2. **GitHub Actions 自动执行**
   - 安装 Node.js 20
   - 安装依赖 (`npm ci`)
   - 构建项目 (`npm run build`)
   - 部署到 GitHub Pages

3. **访问在线应用**
   - URL: https://zuoshiyue.github.io/where-winds-meet-gear-analyzer/
   - 部署完成后约 1-2 分钟可访问

### 查看部署状态

1. 访问 GitHub 仓库
2. 点击 "Actions" 标签
3. 查看 "Deploy to GitHub Pages" 工作流
4. 绿色 ✓ 表示部署成功

## 🛠️ 本地开发

### 环境要求

- Node.js 18+

```javascript

  base: '/where-winds-meet-gear-analyzer/',  // GitHub Pages 路径

```

**注意：** 如果你的 GitHub 用户名不是 `zuoshiyue`，需要修改 `base` 配置：

```javascript

```

### GitHub Pages 设置

1. 访问仓库 Settings
2. 点击 "Pages"
3. Source 选择 "GitHub Actions"
4. 保存后会自动使用 Actions 部署

## 🔧 故障排查

### 部署失败

**检查点：**
1. 查看 GitHub Actions 日志
2. 确认 `web/package.json` 存在
3. 确认 `web/vite.config.js` 配置正确

### 页面空白

**可能原因：**
1. `base` 路径配置错误
2. 路由配置问题
3. 浏览器缓存

**解决方法：**
1. 检查浏览器控制台错误
2. 清除缓存后重试
3. 确认 `vite.config.js` 中的 `base` 配置

### OCR 无法加载

**原因：** Tesseract.js 语言数据加载失败

**解决方法：**
1. 检查网络连接
2. 使用 CDN 加载语言数据
3. 本地部署语言数据文件

## 📊 性能优化

### 代码分割

已在 `vite.config.js` 中配置：

```javascript
rollupOptions: {
  output: {
    manualChunks: {
      'vendor-vue': ['vue'],
      'vendor-element': ['element-plus'],
      'vendor-ocr': ['tesseract.js'],
    },
  },
}
```

### 懒加载

- Tesseract.js worker 按需创建
- 组件按需加载

### 本地存储

- 使用 IndexedDB (localforage)
- 数据持久化在浏览器

## 🌐 自定义域名 (可选)

### 配置步骤

1. 在仓库 Settings → Pages → Custom domain
2. 输入你的域名
3. 在域名 DNS 添加 CNAME 记录：
   ```
   CNAME zuoshiyue.github.io
   ```

### 更新 vite.config.js

```javascript
base: '/',  // 使用根路径
```
